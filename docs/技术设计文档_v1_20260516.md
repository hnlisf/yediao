## 2. 数据库设计

### 2.1 用户表 (users)
```sql
CREATE TABLE users (
    id SERIAL PRIMARY KEY,
    phone VARCHAR(20) UNIQUE NOT NULL,
    nickname VARCHAR(50),
    avatar_url TEXT,
    fishing_age INT DEFAULT 0,           -- 钓龄
    frequent_spots TEXT[],               -- 常驻钓点
    skilled_fish TEXT[],                 -- 擅长鱼种
    privacy_settings JSONB DEFAULT '{}', -- 隐私设置
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2.2 钓位表 (spots)
```sql
CREATE TABLE spots (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    latitude DECIMAL(10, 8) NOT NULL,    -- 纬度
    longitude DECIMAL(11, 8) NOT NULL,   -- 经度
    fish_species TEXT[],                 -- 鱼种
    water_depth VARCHAR(50),             -- 水深
    water_temp VARCHAR(50),              -- 水温
    bait TEXT[],                         -- 饵料
    fishing_methods TEXT[],              -- 钓法
    photos TEXT[],                       -- 照片URL
    creator_id INT REFERENCES users(id),
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2.3 钓位收藏表 (spot_favorites)
```sql
CREATE TABLE spot_favorites (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    spot_id INT REFERENCES spots(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(user_id, spot_id)
);
```

### 2.4 钓位评论表 (spot_reviews)
```sql
CREATE TABLE spot_reviews (
    id SERIAL PRIMARY KEY,
    spot_id INT REFERENCES spots(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    rating INT CHECK (rating >= 1 AND rating <= 5),
    content TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.5 AI识鱼记录表 (fish_recognitions)
```sql
CREATE TABLE fish_recognitions (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    image_url TEXT NOT NULL,
    fish_name VARCHAR(100),              -- 鱼名
    scientific_name VARCHAR(100),        -- 学名
    habits TEXT,                         -- 习性
    protection_level VARCHAR(50),        -- 保护级别
    is_protected BOOLEAN DEFAULT FALSE,  -- 是否保护鱼类
    confidence DECIMAL(5, 4),            -- 置信度
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.6 动态表 (posts)
```sql
CREATE TABLE posts (
    id SERIAL PRIMARY KEY,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    images TEXT[],
    likes_count INT DEFAULT 0,
    comments_count INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2.7 动态点赞表 (post_likes)
```sql
CREATE TABLE post_likes (
    id SERIAL PRIMARY KEY,
    post_id INT REFERENCES posts(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(post_id, user_id)
);
```

### 2.8 动态评论表 (post_comments)
```sql
CREATE TABLE post_comments (
    id SERIAL PRIMARY KEY,
    post_id INT REFERENCES posts(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT NOW()
);
```

### 2.9 约钓活动表 (fishing_events)
```sql
CREATE TABLE fishing_events (
    id SERIAL PRIMARY KEY,
    creator_id INT REFERENCES users(id) ON DELETE CASCADE,
    spot_id INT REFERENCES spots(id),
    title VARCHAR(200) NOT NULL,
    description TEXT,
    start_time TIMESTAMP NOT NULL,
    end_time TIMESTAMP,
    max_participants INT DEFAULT 10,
    current_participants INT DEFAULT 1,
    status VARCHAR(20) DEFAULT 'open',   -- open/closed/cancelled
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);
```

### 2.10 约钓报名表 (event_participants)
```sql
CREATE TABLE event_participants (
    id SERIAL PRIMARY KEY,
    event_id INT REFERENCES fishing_events(id) ON DELETE CASCADE,
    user_id INT REFERENCES users(id) ON DELETE CASCADE,
    status VARCHAR(20) DEFAULT 'joined', -- joined/cancelled
    created_at TIMESTAMP DEFAULT NOW(),
    UNIQUE(event_id, user_id)
);
```

## 3. API接口设计

### 3.1 认证模块
- `POST /api/v1/auth/send-code` - 发送验证码
- `POST /api/v1/auth/login` - 手机号登录
- `POST /api/v1/auth/refresh` - 刷新Token
- `GET /api/v1/auth/profile` - 获取用户信息
- `PUT /api/v1/auth/profile` - 更新用户信息

### 3.2 钓位模块
- `GET /api/v1/spots` - 钓位列表 (支持附近搜索)
- `GET /api/v1/spots/:id` - 钓位详情
- `POST /api/v1/spots` - 新增钓位 (UGC)
- `POST /api/v1/spots/:id/favorite` - 收藏钓位
- `DELETE /api/v1/spots/:id/favorite` - 取消收藏
- `GET /api/v1/spots/:id/reviews` - 钓位评论
- `POST /api/v1/spots/:id/reviews` - 发表评论

### 3.3 AI识鱼模块
- `POST /api/v1/fish/recognize` - 图片识别
- `GET /api/v1/fish/history` - 识别历史
- `GET /api/v1/fish/:id` - 识别详情

### 3.4 社区模块
- `GET /api/v1/posts` - 动态列表
- `POST /api/v1/posts` - 发布动态
- `POST /api/v1/posts/:id/like` - 点赞
- `POST /api/v1/posts/:id/comments` - 评论
- `GET /api/v1/posts/:id/comments` - 评论列表

### 3.5 约钓模块
- `GET /api/v1/events` - 约钓列表
- `POST /api/v1/events` - 发起约钓
- `GET /api/v1/events/:id` - 约钓详情
- `POST /api/v1/events/:id/join` - 报名
- `POST /api/v1/events/:id/cancel` - 取消报名

## 4. 技术栈

- **后端**: NestJS + TypeScript + Node.js 20
- **数据库**: PostgreSQL 15 + TypeORM
- **缓存**: Redis (ioredis)
- **认证**: JWT + Passport
- **API文档**: Swagger (@nestjs/swagger)
- **地图**: 高德地图SDK (Web/移动端)
- **AI**: 百度EasyDL API
- **IM**: 融云IM PaaS
- **存储**: 阿里云OSS (模拟)
- **部署**: Docker Compose

## 5. 部署架构

```
Docker Compose:
├── app (NestJS + Next.js)
├── postgres (PostgreSQL 15)
└── redis (Redis 7)
```

单容器精简部署，适合MVP阶段低成本运行。
